# ZenPass 自動化測試參考

## 測試觸發指南

### API 健康檢查
```
bash scripts/health-check.sh [url] [token]
```
測試：Auth 安全、Dashboard、Users、Classes、Bookings、Payments

### 資料完整性檢查
```
bash scripts/data-integrity.sh [url]
```
驗證：參考編號格式（CL-/US-/ZP-）、關聯完整性、金額/狀態取捨

### 生成報告
```
python3 scripts/generate-report.py
```
輸出：`test-reports/report-YYYYMMDD-HHMMSS.md`

## 常見問題

### Auth
- RequireAdmin 用 hardcode email/name check，唔係 role column
- 冇 `role` 欄位，用 `is_coach` 代替

### 參考編號
- 課程: `CL-YYYYMMDD-XXXX` (classes.class_reference)
- 用戶: `US-YYYYMMDD-XXXX` (users.user_reference)
- 預約: `ZP-YYYYMMDD-XXXX` (bookings.booking_reference)
- Generated by: `generate_reference()` in `src/services/utils.js`

### 狀態選項
- Booking status: `pending_payment`, `confirmed`, `cancelled`, `no_show`, `completed`
- Payment status: `pending`, `paid`, `refunded`, `failed`
- Class status: `active`, `inactive`
- Difficulty: `beginner`, `intermediate`, `advanced`

## 性能測試重點

1. **首屏載入時間**（< 2s）
2. **API 回應時間**（每個 < 500ms）
3. **CSS/JS 壓縮與快取**
4. **圖片優化**（WebP, lazy loading）
5. **Font 載入**（font-display: swap）
