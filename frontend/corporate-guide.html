1.2 Credit 運作

  • 每月自動分配 — 每月 1 號 Reset，未用即過期

  • 離峰／一般時段：12 cr／堂

1.3 設定員工上限（可選）

Admin 可以為每位員工設定每月使用上限，例如每人每月 100 Credits。唔 set 就係冇上限。

💰 三、付費邏輯（自動混合）

系統自動分配：公司 Credits 先俾 → 唔夠就用員工自己嘅 Credits 補。

🟢 情況 A：公司夠 + 未超每月上限 一堂 12 Credits → 公司俾 12，員工自己俾 0 ✅
🟡 情況 B：公司夠但每月上限已滿（上限 10/月） 一堂 12 Credits → 公司俾 0，員工自己俾 12 ✅
🟠 情況 C：公司剩 5 Credits 一堂 12 Credits → 公司俾 5，員工自己俾 7 ✅
🔴 情況 D：公司 0 + 自己都唔夠 ❌ 需先購買 Credits 先可以繼續預約。

員工喺預約頁可以見到：
🏢 公司剩餘 Credits　| 📊 本月已用／上限　| 💰 自己 Credits 餘額

🔄 四、每月 Reset 機制

每月 1 號凌晨，系統自動處理：

  • ✅ credit_used 歸零（公司池 Reset）
  • ✅ 員工 monthly_credit_used 歸零
  • ✅ 上個月未用嘅 Credits 自動到期消失
  • ✅ Audit log 記錄 expired 數量

點解要咁做？

  • 🎯 員工有 incentive 去 Book 堂（Use it or lose it！）
  • 📊 公司 Budget 可預測，每月固定支出
  • 🚫 唔怕囤 Credits 唔用（你而家 0 收入就係因為冇 expiry）
  • 🏆 ClassPass 企業版都係呢個模式

5.2 員工離職處理

Admin 喺後台將員工標記為 inactive 即可。該員工唔可以再用公司 Credits 預約。

❓ 六、常見問題

員工離職點處理？ Admin 可以喺後台將員工標記為 inactive，該員工即無法使用公司 Credits。
員工自己嘅 Credits 同公司 Credits 點分？ 完全分開兩個池。公司 Credits 用公司池，個人 Credits 用自己戶口。Hybrid 模式會自動混合俾錢。
Reset 之後未用嘅 Credits 去邊？ 自動到期消失。每月 allocation 係「呢個月俾你咁多，用唔晒冇得留」。Audit log 有記錄 expired 數量。
可唔可以提早 Reset？ 可以。Admin 後台有「立即 Reset」掣，隨時可以手動 trigger。
員工睇唔睇到公司仲有幾多 Credits？ 睇到。預約時揀「公司 Credits」會顯示公司名 + 剩餘 Credits + 每月用量。
Billing cycle 可以自訂嗎？ 可以。建立企業帳戶時揀選：每月／每季／每年。Reset 日期會對應 billing cycle。

⚙️ 七、系統流程

▼ 每月 1 號
→ credit_used = 0（公司池 reset）
→ monthly_credit_used = 0（員工用量 reset）
→ audit_log 記錄 expired credits
▼ 員工預約
→ 檢查公司池夠唔夠
→ 檢查員工每月上限
→ 自動混合公司 + 個人 Credits
→ Audit log + blockchain 記錄
▼ 每日 Scheduler（每 15 分鐘）
→ 檢查到期 Reset
→ 自動處理
